# Simdaas Autonomy Community Code of Conduct

## Our Pledge

At **Simdaas Autonomy Private Limited**, we are committed to fostering an open, inclusive, and professional environment for everyone contributing to our embedded systems and autonomy software.

We pledge to make participation in our community a **harassment-free experience** for all individuals, regardless of:

- Age  
- Body size  
- Disability  
- Ethnicity  
- Gender identity and expression  
- Experience level  
- Education  
- Socio-economic status  
- Nationality  
- Personal appearance  
- Race  
- Religion  
- Sexual identity and orientation  

---

## Our Standards

We are committed to building a **respectful, collaborative, and engineering-focused environment**.

### Positive Behavior

Examples of behavior that contribute to a positive community:

- Communicating respectfully and professionally  
- Using inclusive and welcoming language  
- Sharing knowledge openly and helping others grow  
- Writing clean, maintainable, and well-documented code  
- Reviewing code constructively (not personally)  
- Accepting feedback gracefully  
- Prioritizing system safety, reliability, and quality  
- Collaborating effectively in issues, pull requests, and discussions  

---

### Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment, discrimination, or hate speech  
- Personal attacks, insults, or derogatory comments  
- Trolling, flaming, or disruptive arguments  
- Sharing private or confidential information without permission  
- Posting irrelevant, spam, or misleading content  
- Unsafe coding practices that may harm systems without proper justification  
- Ignoring project guidelines intentionally  
- Excessive or unhelpful interference in discussions  

---

## Engineering Responsibility (Important for Embedded Systems)

Since this repository involves **embedded systems, hardware interaction, and autonomy**, contributors must:

- Follow safe coding practices (memory safety, concurrency, hardware protection)  
- Avoid introducing code that can damage hardware or create unsafe behavior  
- Clearly document assumptions, limitations, and risks  
- Test changes before submitting pull requests  
- Respect real-time constraints and system reliability  

---

## Our Responsibilities

Project maintainers and core contributors are responsible for:

- Enforcing this Code of Conduct fairly and consistently  
- Reviewing contributions objectively  
- Providing constructive feedback  
- Taking appropriate action against violations  
- Maintaining code quality and system integrity  

Maintainers have the right to:

- Edit or remove contributions (code, issues, comments)  
- Reject pull requests that do not meet standards  
- Temporarily or permanently ban contributors for violations  

---

## Reporting Issues

If you experience or witness unacceptable behavior, please report it:

Email: **support@simdaas.com**

All reports will be:

- Handled confidentially  
- Reviewed seriously and promptly  
- Investigated fairly  

---

## Enforcement Policy

We follow a **progressive enforcement approach**:

1. Warning – Explanation of violation and expected behavior  
2. Temporary restriction – Limited participation or access  
3. Permanent ban – For repeated or severe violations  

Severe violations may result in **immediate removal without warning**.

---

## Scope

This Code of Conduct applies to:

- GitHub repositories  
- Issues and pull requests  
- Discussions and comments  
- Documentation and code reviews  
- Any official Simdaas Autonomy community platform  

It also applies when representing the project in public spaces.
