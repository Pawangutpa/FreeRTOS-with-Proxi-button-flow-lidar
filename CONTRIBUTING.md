# Contributing Guidelines

Thank you for your interest in contributing to **Simdaas Autonomy Private Limited** projects 🚀

This repository is built through the efforts of engineers and contributors who are passionate about **embedded systems, autonomy, and reliable software**. We welcome contributions from everyone — whether it's bug fixes, new features, or improvements.

---

## How to Contribute

The best ways to contribute are:

- Create an **Issue** to report bugs or suggest ideas  
- Submit a **Pull Request (PR)** with improvements or new features  
- Improve documentation  
- Help review other contributions  

---

## General Guidelines

Please follow these basic principles when contributing:

- Be respectful and professional  
- Keep discussions constructive and focused  
- Understand that different approaches and ideas are welcome  
- Follow the project's structure and coding style  

---

## Creating Issues

When creating an issue:

- Use a **clear and meaningful title**  
- Describe the problem or idea in detail  
- Include relevant information:
  - Board / hardware used (ESP32, STM32, etc.)  
  - Firmware version  
  - Logs or error messages  
- Add steps to reproduce the issue (if applicable)  

---

## Creating Pull Requests

When submitting a PR:

- Ensure your code is **tested and working**  
- Keep PRs **small and focused** (avoid huge changes in one PR)  
- Clearly describe:
  - What you changed  
  - Why you changed it  
- Link related issues (if any)  

---

## Code Style Guidelines

For embedded systems development:

- Write **clean and readable code**  
- Use meaningful variable and function names  
- Follow consistent formatting  
- Comment important logic (especially hardware-related code)  
- Avoid unnecessary complexity  

---

## Code Sharing Rules

- Use **code blocks** when sharing snippets  
- Avoid pasting very large code directly (>30–50 lines)  
  - Use GitHub files or external gists instead  
- Share only relevant code to the issue  

---

## Embedded Systems Best Practices

Since this project involves **hardware and real-time systems**, contributors must:

- Ensure code does not damage hardware  
- Handle memory carefully (avoid leaks, overflows)  
- Consider timing and concurrency (FreeRTOS, interrupts, etc.)  
- Validate sensor data and edge cases  
- Test on real hardware whenever possible  

---

## Communication Guidelines

- Be friendly and patient  
- Avoid aggressive or dismissive language  
- Respect different skill levels  
- Ask questions clearly  

---

## What Not to Do

- Do not submit untested or broken code  
- Do not spam or create duplicate issues  
- Do not ignore project structure or guidelines  
- Do not include unrelated changes in a PR  

---

## Review Process

- Maintainers will review your PR  
- Feedback may be provided — please address it constructively  
- Changes may be requested before merging  
- Not all contributions are guaranteed to be accepted  

---

## Final Note

> Good code builds products.  
> Great collaboration builds companies.

We appreciate your contribution to **Simdaas Autonomy** 💡
